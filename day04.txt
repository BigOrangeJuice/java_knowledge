1.面向过程和面向对象的区别【耦合度】

	* 面向过程：耦合度高，拓展性差；关注实现的具体过程，因果关系【集成显卡】【蛋炒饭】

		优点：对于业务逻辑比较简单的程序，可以达到快速开发，前期投入较低

		缺点：1> 难以解决业务逻辑比较复杂的程序

		      2> 另外面向过程的方式会导致软件元素之间的【耦合度】过高，牵一发而动全身

		      3>且无独立体概念，所有无法达到组件复用

	* 面向对象：耦合度差，拓展性强；关注对象【独立体】能完成那些功能【独立显卡】【盖饭】

		优点：耦合度低，扩展力强，更容易解决现实世界当中更复杂的业务逻辑，组件复用性强

		缺点：前期投入高，需要进行独立体的抽取，大量的系统分析与设计

	* 蛋炒饭与盖饭：

		蛋炒饭的优点是饭菜混搭在一起，味道香，
		
			但缺点是没有单独的一个对象【独立体】，很难再添加其他的饭菜，且如果不想要其中的一样食品也很难剥离

		盖饭的优点是饭菜分开，随意搭配，且随时都可以添加别的菜谱，缺点自然是分开后味道不是那么浓厚

3.类和对象

	* 类：
	
		—— 在现实世界中并不存在，是一个模板，是一个概念，是人类大脑思考抽象的结果

		—— 代表了一类事物

		—— 在现实世界中，对象A与对象B之间具有共同特征，进行抽象总结出一个模板，这个模板便是类

	* 对象：

		—— 现实世界中存在的个体

	* 软件开发的过程：

		1> 程序员先观察现实世界，从现实世界当中寻找对象

		2> 寻找到N个对象之后，发现对象们的共同特征

		3> 程序员在大脑中形成了一个模板【类】

		4> Java程序员通过java代码来表述一个类

		5> Java程序中有了类的定义

		6> 然后通过类可以创建对象

		7> 有了对象，让对象之间协作起来形成一个系统

	* 类 ----【实例化】--->对象；对象-----【抽象化】---->类；对象也叫实例(instance)

	* 类描述 状态 + 动作

		—— 状态：名字、身高、性别。。。。
		—— 动作：唱、跳、rap。。。。

		—— 状态：类的属性
		—— 动作：类的方法

		—— 类{
			属性 ;	//描述对象的状态信息
			方法 ;	//描述对象的动作信息
			}

4.类的定义 

	*语法结构：
		
		[修饰符列表] class 类型{

			属性 ;

			方法 ;

		}

	* 类中的实例变量可以初始化，使其自带数值

	* 若没有初始化，则会自带默认值

	* 属性和变量的区别：

		类里定义的数据成员称为属性，属性可不赋初值，若不赋初值则JAVA会按上表为其添加默认值；【思考：为什么程序会给属性附默认值】
		
		方法里定义的数据成员称为变量，变量在参与运算之前必须赋初值

5.引用数据类型：String、System、或自己自定义的类

	* 有默认值，且默认值为 null 空值

6.实例化对象：new 类名() ;

7.new运算符的作用是创建对象，在JVM的堆内存中开辟新的内存空间

8.对象：new在堆内存中开辟的内存空间

	* 实例变量必须下创建对象，通过引用.的方式访问，不能直接使用类名.的方式访问

9.引用：一个变量，可能是局部变量、成员变量，其保存了另一个java对象的内存地址
		
	只有引用才能够找到对象

10.由于java里没有指针，要想访问堆内存中对象内部的实例变量，要用引用

	成员变量：在类内方法外定义的变量

		—— 分类：实例变量(无static修饰)和类变量(static修饰)

	* 访问实例变量的语法格式：

		读取数据：引用.变量名

		修改数据：引用.变量名 = 值

11.实践：

	* 在深入使用其他类时，例如有主类，A类，B类，而此时B类在A类中有被当做引入数据类型调用

	  当主类通过new给A类在堆区开辟了一个内存空间后，要想使用到A类中的B类，需要给B类也通过new开辟一个内存空间

	  否则会报错，地址为null

	* 写成了单个学生信息存储系统

12.线程：操作系统能够进行运算调度的最小单位

13.报错：java.lang.NullPointerException

	* 编译通过，运行出错：空引用访问“实例”相关的数据一定会出现空指针异常

	* “实例”相关的数据：这个数据访问时必须有对象的参与
